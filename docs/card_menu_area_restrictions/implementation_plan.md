# 実装計画: エリアごとのカードメニュー出し分け

## 目的
カード（ポケモン）が置かれている場所（アクティブ、ベンチなど）によって、クリック時に表示されるメニューの内容と実行できるアクションを制限・変更します。

## 要求仕様
1. **アクティブエリア**
   - ダメージの変化
   - 特殊状態（状態異常）の付与・回復

2. **ベンチエリア**
   - ダメージの変化
   - **特殊状態のボタンは非表示（付与不可）**
   - **ベンチに移動・存在する場合は特殊状態をすべて回復（クリア）する** 

## ユーザーレビュー依頼
> [!IMPORTANT]
> ベンチでの「特殊状態が全て回復し、付与もされない」仕様について、「ベンチに下がった瞬間に状態異常が消える」システム的な連動を `Board.tsx` のカード移動ロジック（`handleDragEnd`）に組み込む方針で進めます。

## 検証計画
- コマンド `npm run dev` でローカルサーバーを起動。
- アクティブにあるカードをクリックし、ダメージと状態異常の操作ができるか確認。
- ベンチにあるカードをクリックし、状態異常のメニューが出ないことを確認。
- アクティブで状態異常にしたカードをベンチにドラッグ移動した際、状態異常が自動で消去されることを確認。
