# 実装完了: エリア別のカードメニュー出し分け

## 変更内容
カードが配置されているエリア（ベンチ、アクティブなど）に応じて表示されるメニューを制限し、状態異常に関するシステム的な動作を制御する機能を実装しました。

### 1. メニューUIの出し分け (`CardMenu.tsx`, `Card.tsx`)
- ベンチ（`bench`）に配置されたポケモンを選択した際は、「状態異常」のメニュー要素（アイコン一覧）が表示されないように修正しました。
- それ以外のエリア（`active` など）では従来通り、全てのダメージカウンター・状態異常アイコンが操作可能です。

### 2. 特殊状態の自動回復 (`Board.tsx`)
- いずれの場所からカードが**ベンチ（`bench`）へ移動（ドラッグ＆ドロップ）した場合**、そのカードが持っている状態異常のデータ (`cnd`) を自動的にリセットし、空配列（未罹患状態）にする処理を追加しました。
- （先に追加済みの）デッキへ戻した際に裏向き（`f: false`）になる機能も引き続き作動します。

## テスト結果
- TypeScript の型チェック（`npx tsc --noEmit`）は正常、エラーなしで通過しました。

## 手動確認のお願い
現在バックグラウンドで `npm run dev` が実行中です（`http://localhost:5173`）。
以下の機能が以前と同じように動作するか、ブラウザでアクセスしてご確認ください：

- [ ] **アクティブ**のカードをクリックしたとき、「ダメージ」と「状態異常」の両方のメニューが表示されるか。
- [ ] **ベンチ**のカードをクリックしたとき、状態異常のメニューが **消えている（表示されない）** か。
- [ ] アクティブにて何らかの状態異常（どく、やけど等）を付加したカードをベンチへドラッグした際、付与されていた状態異常アイコンが **自動的に消えるか**。
